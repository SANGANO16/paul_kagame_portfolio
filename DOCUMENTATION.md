Paul Kagame Portfolio Website – Report
Done by: SANGANO Jean Claude
Reg No: 25RP02065

1. Website Purpose
The Paul Kagame Portfolio Website is an educational web project developed to present information about the life, leadership, and achievements of the President of Rwanda. The website organizes content in a clear and interactive way, allowing users to explore sections such as biography, leadership, achievements, news, events, and resources.

The project also demonstrates practical front-end development skills by creating a responsive and user-friendly website with interactive features.

2. Tools and Technologies Used
The website was developed using the following technologies:

HTML5 for structuring web pages

CSS3 for styling and layout (Flexbox, Grid, animations)

JavaScript (ES6+) for interactivity and dynamic functionality

Browser APIs used:

localStorage for storing user data (comments, likes, visits)

sessionStorage for tracking session activity

Intersection Observer API for scroll animations

Additional tools:

Font Awesome for icons

Google Fonts for typography

3. Key Features Implemented
Responsive Design: Works on desktop, tablet, and mobile devices

Dark/Light Theme Toggle: Saves user preference

Interactive News Section: Includes filtering, “Read More,” and “View More”

Comment System: Users can post comments stored in localStorage

Like System: Users can like/unlike articles with saved counts

Visitor Tracking: Tracks visits and user activity

Scroll Features: Scroll-to-top button and reveal animations

Additional Pages: Events and Resources pages for extended content

4. Challenges Faced and Solutions
Data Storage Without Backend:
Storing user interactions without a database was challenging.
Solution: Used localStorage to store and retrieve data.

Managing Multiple Features:
Combining several interactive features increased complexity.
Solution: Organized code into reusable functions and structured storage keys.

Responsive Layout Issues:
Ensuring proper display across devices required adjustments.
Solution: Used Flexbox, Grid, and media queries.

Performance of Animations:
Scroll-based animations affected performance.
Solution: Implemented Intersection Observer API for efficiency.

Conclusion
This project demonstrates the ability to build a responsive and interactive website using modern front-end technologies while addressing real-world development challenges without backend support.
